## 😫 痛点直击：由于“传统”，所以“沉重”

对于一线地质工作者来说，**数字地质调查系统 (DGSS)** 是我们的老朋友，但也是个“暴躁老哥”：

1.  **软件太重**：查看 `.ta`、`.la` 格式数据，往往需要依赖 GeoMap 或 MapGIS 等大型桌面软件，启动慢、操作繁琐。
2.  **门槛太高**：想要筛选特定岩性的数据？你可能得去写 SQL 语句，即使你是地质专家，也不该被代码难住。
3.  **效率太低**：野外采集的数据难免有误，人工逐条检查坐标、修正错别字，耗时耗力且容易遗漏。
4.  **数据孤岛**：数据躺在硬盘里只是“死数据”，无法快速被检索和利用。

**我们需要一个轻量、智能、能“听懂人话”的工具。**

---

## 💡 解决方案：轻量化 + 本地AI

本项目是一个 **Web 化的 DGSS 数据交互平台**，不仅能直接解析野外手图数据，更革命性地集成了 **Ollama 本地大模型**。

它让你的地质数据管理变得前所未有的简单：
*   **扔掉重型软件**：浏览器直接打开，秒级加载。
*   **数据不出门**：本地 AI 运行，无需联网，严格保障涉密数据安全。
*   **专家级助手**：像聊天一样管理数据。

## 🚀 核心功能亮点

### 1. 📂 秒解 DGSS 格式，一目了然
原生支持解析 DGSS 特有的 SQLite 存储格式，无论是**点状地质体 (.ta)**、**线状 (.la)** 还是**面状 (.pa)**，拖入即看。自动关联字典，把晦涩的代码（如 `D_CODE`）变成人话（如 `地层代号`）。

![文件说明](使用说明图片/数据库文件说明.jpg)
*(▲ 图：深度适配 DGSS 数据结构，自动解析关键字段)*

### 2. 🧠 AI 赋能，让数据“活”起来
这是本项目的杀手锏！内置 AI 助手（基于 Qwen 等模型），让地质工作自动化：

*   **🗣️ 自然语言查询**：
    *   *“帮我找出所有花岗岩的记录”* —— AI 自动帮你查。
*   **✍️ 智能填充与纠错**：
    *   *“检查这批数据的坐标格式对不对”* —— AI 帮你质检。
    *   *“根据路线描述生成一个合理的观测点记录”* —— AI 辅助填表。

### 3. ⚡ 批量操作，效率翻倍
支持多选记录进行批量修改。配合 AI，你可以一键完成“将所有‘细砂岩’备注修正为‘含砾细砂岩’”这种复杂操作。

## 📖 快速上手指南

使用本工具非常简单，只需三步：

### 第一步：环境配置
确保电脑安装了 **Python 3.12+**。安装依赖：
```bash
pip install -r requirements.txt
```

### 第二步：召唤 AI (安装 Ollama)
下载并安装 [Ollama](https://ollama.com/)，在终端拉取一个中文能力强的模型（推荐 Qwen2.5）：
```bash
ollama pull qwen2.5:14b
```

### 第三步：启动飞行
双击项目根目录下的 `start.bat`，浏览器会自动打开操作界面。
